# 젬대리 비주얼 자산 깃허브 배포 계획

## 🎯 목표
현재 로컬에만 존재하는 젬대리의 다양한 표정 이미지들을 깃허브 저장소에 업로드하고, 어디서든 접근 가능한 **Raw GitHub URL**을 확보하여 문서 및 채팅 품질을 높입니다.

## 📌 현재 상태 분석
- **저장소:** `https://github.com/imejaim/20_agent_charactor` (master 브랜치)
- **대상 파일:** `.agent/skills/JEM_Daeri/` 폴더 내 PNG 2종 및 SVG 15종
- **문제점:** 현재 로컬 상대 경로로 설정되어 있어, 특정 환경(채팅창 등)에서 이미지가 깨질 수 있음.

## 🛠️ 구현 단계
1. **Git Commit & Push:** 로컬의 이미지 파일들을 저장소에 업로드합니다.
2. **URL 추출:** 각 이미지에 대응하는 `raw.githubusercontent.com` 주소를 생성합니다.
3. **문서 업데이트:** `JEM_Daeri.md`의 이미지 링크를 생성된 깃허브 URL로 교체(또는 병기)합니다.

## 💡 대안 제시 (Options)

### Option 1: 로컬 상대 경로 유지 + 업로드만 수행 (추천 🌟)
- **장점:** 저장소의 이식성이 유지되며, 깃허브 웹 UI에서도 잘 보입니다.
- **특징:** 제가 답변할 때만 깃허브 URL을 사용하여 이미지를 보여드립니다.

### Option 2: 문서 내 모든 링크를 깃허브 Raw URL로 전면 교체
- **장점:** 어떤 환경에서 문서를 열어도 이미지가 절대 깨지지 않습니다.
- **단점:** 오프라인 환경이나 깃허브 주소가 바뀔 경우 링크를 수정해야 합니다.
